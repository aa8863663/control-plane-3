MTCP Methodology

MTCP stands for Multi‑Turn Constraint Persistence. The framework tests whether a model can continue following an explicit rule after it has already violated that rule and has been corrected.

Framework overview

The benchmark is built around repeated boundary checks and follow-up correction. It measures whether a model can recover and persist rather than whether it can pass a single-shot prompt.

  • Main probes
    Canonical benchmark set used across four temperatures.
  • Control probes
    Smaller fresh set designed to reveal degradation outside the main evaluation surface.
  • Temperature profile
    Used to understand variance and classify model instability.
  • Grade
    Derived from average main benchmark performance.

Five vectors

The platform groups evaluation patterns into core operational vectors without publishing the underlying probes.

  • Instruction persistence
    Does the model continue following the rule after correction?
  • Boundary integrity
    Does it stay inside explicit response boundaries?
  • Format discipline
    Can it preserve required structure and output constraints?
  • Control degradation
    How much performance survives on fresher probe variants?
  • Temperature stability
    Does behaviour shift materially with higher sampling variance?

Grading scale

Grades are assigned from the average main benchmark score.

Probe structure

Public documentation explains the evaluation logic but never discloses the private probe texts.

Probe content is intentionally withheld. The platform explains the framework, grading, vectors, and comparative logic while keeping the evaluation set private.
