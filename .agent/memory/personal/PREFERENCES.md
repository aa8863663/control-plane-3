# Communication Style
- Direct and concise
- Data first, then interpretation
- No em dashes
- No semicolons
- No bullet points in prose
- Short sentences
- Active voice
- Never use: can, may, just, very, really, literally, actually, certainly, probably, basically
- Always A. Abby pseudonym
- Technical depth without jargon
- Specific numbers over vague claims
- No hashtags in academic writing
- Hashtags only on LinkedIn posts

# Research Style
- Empirical claims require database evidence
- Theoretical claims require framework reference
- All numbers from verified source
- Falsifiable predictions only
- Cite Paper I, II, III for established findings

# Platform Technical Notes
- Python 3.10 on Fly.io Frankfurt
- psycopg2 with %s placeholders only
- Neon PostgreSQL EU Central
- Bedrock requires eu. prefix
- Bedrock max 1 concurrent model
- Always check fetchone() for None
- recovery_latency stored as TEXT, cast to FLOAT for AVG()
