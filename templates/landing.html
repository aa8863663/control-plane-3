🔬 Key Paper Findings — Novel Cross-Architecture Results

1. Open source beats paid Claude on MTCP. LLaMA 3.3 70B averages 75.6% (grade C). Claude Sonnet 4 — which costs ~8× more per token than Haiku — averages just 68.1% (grade D). The entire Claude family clusters between 65–69% regardless of model tier or price point.

2. Temperature has almost no effect on Claude. Haiku spans only 2% across 4 temperatures. Sonnet at T=0.5 and T=0.8 scored identically (67.9%). This is systematic failure behaviour, not random sampling noise — a distinct failure mode from the LLaMA family.

3. Higher temperature hurts LLaMA but not Claude. LLaMA 3.1 8B drops 7 points from T=0.2 to T=0.8. LLaMA 3.3 70B stays flat (74–77%). Differential temperature sensitivity suggests fundamentally different failure modes between architectures.

4. No model passes MTCP — highest grade is C. Best result: LLaMA 3.3 70B at T=0.0 — 76.6%, 59 hard stops. Still only a C. No production model reliably follows corrected instructions at scale. Substantial room for industry improvement.

The MTCP Protocol — Three-Turn Probe Architecture
T1
Initial Probe
Prompt with embedded explicit constraint. Response evaluated against specification.
T2
Correction
Structured correction identifying violation. Recovery latency tracked from this turn.
T3
Reinforced
Final correction with reinforced language. Persistent failure triggers enforcement.
HS
Hard Stop
Automated termination. SHA-256 verified audit trail. Regulator-ready export.
