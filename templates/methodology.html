Methodology

How MTCP Works

A structured, reproducible, provider-agnostic evaluation framework for post-correction constraint persistence.

What MTCP Measures
MTCP measures post-correction reliability — whether a language model maintains compliance with an explicit constraint after that constraint has been corrected. This is distinct from single-turn instruction following. Existing benchmarks test whether a model can follow an instruction once. MTCP tests whether it continues to follow it after explicit correction within the same conversation.
Three-Turn Probe Structure
Each probe is a deterministic three-turn sequence.

Turn 1 — Task with embedded constraint. Compliance assessed automatically.
Turn 2 — If violation detected: corrective instruction restating the constraint. Compliance re-assessed.
Turn 3 — If second violation: reinforcement correction. Terminal state recorded.

A probe is classified as COMPLETED (passed) if compliance is achieved and maintained. It is classified as SAFETY_HARD_STOP (failed) if the model does not return to compliance by turn 3.

Grading Scale

Research Registration
This benchmark is registered on OSF under DOI 10.17605/OSF.IO/DXGK5. The evaluation methodology covers three integrated papers: Paper I (MTCP Benchmark), Paper II (Identity-Gate Satiation and Latent Attractor Dynamics), and Paper III (Sigma-Forensics: A Transcript-Based Safety Audit Framework).
